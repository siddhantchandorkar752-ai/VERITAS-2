## 🧠 Core Architecture

Instead of relying on a single AI model to "guess" an answer, VERITAS-Ω simulates a rigorous courtroom debate using three independent agents and a deterministic judge.

1. **Multi-Agent Debate Framework**: 
   - **Pro Agent**: Actively searches for evidence supporting the claim.
   - **Con Agent**: Actively searches for evidence contradicting the claim.
   - **Adversarial Agent**: Audits the reasoning of both, identifying logical fallacies, semantic drift, and source bias.
2. **Deterministic Mathematical Aggregation**: 
   - The final verdict is *not* generated by an LLM. It is mathematically calculated using a bounded confidence formula: `Conf = (Pro - Con + GraphDiff)/2 - Penalty`.
   - The Adversarial Agent's findings directly penalize the final confidence score, preventing false consensus.
3. **Uncertainty Quantification**: 
   - The system mathematically separates **Epistemic Uncertainty** (missing data/adversarial gaps) from **Aleatoric Uncertainty** (systemic disagreement), displaying this exact breakdown to the user.
4. **Evidence Topology Graph**: 
   - Uses `NetworkX` and `PyVis` to physically map the relationships between the original claim, the extracted sub-claims, and the independent source documents.

---

## ⚡ The Execution Pipeline

When a user submits a claim (e.g., *"The UN mandates mandatory imprisonment for XYZ"*), the following happens in real-time:

1. **Claim Grounding**: The engine strips emotional bias and breaks the input down into atomic, verifiable sub-claims.
2. **Independent Retrieval**: The system queries domain-specific databases (ArXiv, Web, Legal Docs) ensuring absolute source diversity. No two agents can reuse the same evidence.
3. **Agent Scoring**: The Pro, Con, and Adversarial agents evaluate the independent evidence and output raw confidence integers.
4. **Synthesized Verdict**: The Python-based `JudgeSystem` executes the aggregation formula to compute the absolute Truth Score (TRUE, FALSE, PARTIALLY_TRUE, or UNCERTAIN).

## 🚀 How to Run

### Option 1: Local Deployment

1. **Clone the repository**
   ```bash
   git clone https://github.com/siddhantchandorkar752-ai/VERITAS-2.git
   cd VERITAS-2
   ```

2. **Install Dependencies**
   ```bash
   pip install -r requirements.txt
   ```

3. **Configure Environment**
   Create a `.env` file in the root directory:
   ```toml
   OPENAI_API_KEY=your_api_key_here
   USE_MOCK_LLM=true  # Set to 'false' to use real OpenAI calls
   ```

4. **Launch the Engine**
   ```bash
   python -m streamlit run ui/app.py
   ```

### Option 2: Cloud Deployment (Streamlit Community Cloud)

1. Go to [share.streamlit.io](https://share.streamlit.io/).
2. Create a **New App** and select this GitHub repository.
3. Set the **Main file path** to `ui/app.py`.
4. Open **Advanced Settings** and paste your `.env` variables into the **Secrets** box.
5. Click **Deploy**. The app will be live globally in under 2 minutes.
